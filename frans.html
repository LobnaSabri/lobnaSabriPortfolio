Projets sur lesquels j'ai travaillé pendant mes études:

  • Software Development Project I

    1. Discription

      2. En première année, au second semestre, nous avons dû réaliser un jeu en java en groupe. Le jeu que nous devions faire était Rummikub. En java, nous étions autorisés à utiliser scenebuilder pour faire le l'interface utilisateur.

    2. Ma contribution

      3. Dans mon groupe, nous n'avons pas divisé les tâches, nous nous sommes mis d'accord étape par étape qui veut ou peut faire quoi. De manière générale, j'ai aidé à l'analyse, qui consiste à créer un contrôleur de domaine. Sur les 4 membres du groupe, j'étais celui qui a fait la le plus codé.

    3. Mon avis

      4. La mission était claire et j'avais déjà joué le jeu auparavant. Le codage m'intéresse beaucoup, alors même si je n'avais pas beaucoup de connaissances, c'était amusant d'avoir une liberté limitée pour une fois.

  • Business Software Project

    1. Discription

      2. En deuxième année de licence, j'ai dû réaliser une application web en groupe. Nous avons reçu un client qui avait une vision et nous avons commencé par élaborer cette vision à l'aide de cas d'utilisation, de wireframes et de maquettes. Une fois que nous étions sur la même ligne de pensée que le client, nous avons pu commencer à réaliser le backend et le frontend (Sapgui5) à l'aide de SAP.

    2. Ma contribution

      3. Comme l'année dernière, nous n'avons pas divisé toutes les tâches, nous avons pris nous-mêmes l'initiative de travailler sur le code, étape par étape. Les seules tâches qui ont été partagées sont les cas d'utilisation, les wireframes et les maquettes. Comme SAP était nouveau pour nous tous, nous étions souvent bloqués, mais heureusement, nous avions les professeurs pour nous aider.

    3. Mon avis

      4. Ce projet m'a appris que la communication est très importante, tant avec le client qu'au sein de votre groupe. Il est également nécessaire de toujours vérifier les besoins du client afin de ne rien oublier.
